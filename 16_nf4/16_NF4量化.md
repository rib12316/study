## 一、直觉版讲解（先建立直觉，再看数学）

### 1.1 问题：4bit 只能表示 16 个数

4bit 量化，本质是用 **16 个固定的数**（量化点）去近似所有可能的权重值。比如：

- 权重 = 0.13 → 找 16 个量化点里最接近的，存它的编号
- 权重 = -0.42 → 同上
- 反量化时，把编号还原成那个量化点的值

**核心问题**：这 16 个量化点该放在哪？放得不同，量化误差就不同。

### 1.2 朴素做法：均匀分布（INT4）

最直觉的放法：**在 [-1, 1] 区间均匀摆 16 个点**。就像尺子上的刻度，等间距：

```
均匀分布（INT4）的 16 个量化点（示意，归一化后）：
-1.0  -0.87  -0.73  -0.60  -0.47  -0.33  -0.20  -0.07   0.07   0.20   0.33   0.47   0.60   0.73   0.87   1.0
 |     |      |      |      |      |      |      |      |      |      |      |      |      |      |     |
 （每两个点间距相等 ≈ 0.13）
```

**均匀量化的盲点**：它假设权重在 [-1,1] 上**均匀分布**——到处一样密。但 LLM 权重**不是均匀分布**的！

### 1.3 关键观察：LLM 权重是正态分布（钟形）

实测 LLM 的权重值分布近似**正态分布（高斯钟形）**：
- **大部分权重聚集在 0 附近**（钟形的峰）
- 离 0 越远，权重越稀少（钟形的尾巴）
- 极少数权重的绝对值很大（两端的极值）

```
权重密度（正态分布，钟形）：
密度
 │      ┌──┐
 │     ╱    ╲
 │    ╱      ╲
 │   ╱        ╲
 │  ╱          ╲
 │_╱____________╲_______ 权重值
-1  -0.5  0  0.5  1
   ↑           ↑
  尾巴(稀疏)   峰(密集)
```

### 1.4 均匀量化为什么浪费

既然权重密集在 0 附近（峰），但均匀量化在 0 附近只放了和别处一样多的点。结果：
- **0 附近**：权重超密集，但量化点不够密 → 大量权重挤在少数几个点，**误差大**（人最多的地方，尺子刻度反而稀）
- **两端（±1）**：权重超稀少，但量化点和别处一样多 → **浪费**（没人的地方，刻度反而密）

**直觉一句话**：均匀量化把"弹药"（16 个点）平均分给所有区域，但权重分布不平均——密集区不够用，稀疏区浪费。

### 1.5 NF4 的设计目的：把量化点"密集放在权重多的地方"

NF4（NormalFloat 4-bit）的核心思想：**既然权重是正态分布，那 16 个量化点也按正态分布的分位数摆**——0 附近摆密一点（因为权重多），两端摆稀一点（因为权重少）。

```
NF4 的 16 个量化点（示意，按正态分位数）：
-1.0  -0.7  -0.5  -0.36 -0.25 -0.17 -0.09  0  0.09  0.17  0.25  0.36  0.5  0.7  0.86  1.0
  |    ||    ||    |||   |||   |||   ||||  | ||||  |||   |||   |||   ||   ||   |    |
  尾巴(点稀)              峰(点密)                    峰(点密)              尾巴(点稀)
```

**对比均匀**：
- 0 附近：NF4 放了 4-5 个点（密集区），均匀只放 2 个 → NF4 在权重最多的地方**精度更高**
- 两端：NF4 只放 2-3 个点（稀疏区），均匀放 4 个 → NF4 不浪费

**设计目的达成**：把有限的 16 个量化点，**按权重的分布密度智能分配**——密集区多给、稀疏区少给，整体量化误差最小化。

### 1.6 一个具体数值对比

假设三个权重值：`w1=0.02, w2=0.10, w3=0.80`（0 附近两个、尾部一个）。

**均匀量化**（间距 ~0.13）：
- w1=0.02 → 最近点 0.07 → 误差 0.05
- w2=0.10 → 最近点 0.07 → 误差 0.03
- w3=0.80 → 最近点 0.73 → 误差 0.07

**NF4 量化**（0 附近密）：
- w1=0.02 → 最近点 0（NF4 有 0 这个点）→ 误差 0.02 ✓ 更准
- w2=0.10 → 最近点 0.09 → 误差 0.01 ✓ 更准
- w3=0.80 → 最近点 0.70 → 误差 0.10（尾部点稀，这里 NF4 稍差）

**净效果**：NF4 在权重密集的 0 附近大幅提升精度（0.05→0.02、0.03→0.01），尾部略损（0.07→0.10）。但因为**密集区的权重数量远多于尾部**，整体期望误差显著降低。这就是 NF4 比均匀 INT4 精度高的根本原因。

> **一句话总结 NF4 的设计目的**：既然 LLM 权重是正态分布，就把 16 个量化点也按正态分布摆，让"弹药"精准投放到权重密集区，整体误差最小。这是**数据感知（data-aware）量化**的经典设计。

## 二、NF4 的数学：量化点怎么算（QLoRA 论文）

### 2.1 正态分布的分位数

正态分布 N(0,1) 的分位数 `q_i`：把概率密度曲线下面积等分成 16 份，每份的分界点就是量化点。

数学上，NF4 用**对称的正态分位数**（论文式 4-7）。因为权重关于 0 对称，NF4 也对称。论文给出的 NF4 16 个量化点（归一化到 [-1,1]）：

```
NF4 量化点（论文固定常量，bitsandbytes 硬编码）：
[-1.0, -0.6962, -0.5251, -0.4391, -0.3439, -0.2520, -0.1630, -0.0842,
  0.0842,  0.1630,  0.2520,  0.3439,  0.4391,  0.5251,  0.6962,  1.0]
```

注意：**0 附近点密集**（-0.0842, 0.0842 离 0 很近），**两端稀疏**（-1.0 到 -0.6962 跨度大）。

> 这 16 个值是**论文预先算好的常量**，不需要运行时计算。bitsandbytes 直接硬编码。vLLM 加载时调 `quantize_4bit(quant_type="nf4")`（`bitsandbytes_loader.py:436`）使用这套点。

### 2.2 归一化：让权重适配 NF4 的 [-1,1]

权重实际范围不是 [-1,1]。NF4 流程：
1. **归一化**：对一组权重，找最大绝对值 `max_abs`，所有权重除以 `max_abs` → 压到 [-1,1]
2. **量化**：归一化后的权重，找最近的 NF4 量化点，存编号（0-15）
3. **存 scale**：`scale = max_abs`（反量化时乘回来）

```
权重 0.42, max_abs=1.5 → 归一化 0.42/1.5=0.28 → 最近NF4点 0.2520 → 存编号11
反量化：编号11 → 0.2520 × scale(1.5) = 0.378（原值0.42，误差0.042）
```

### 2.3 group_size=64：局部归一化

第14讲提到 NF4 用 group_size=64。意思：不是对整个权重矩阵求一个 max_abs，而是**每 64 个权重一组，各自求 max_abs 归一化**。这样每组的权重范围更窄，归一化后更贴近 [-1,1]，量化点利用更充分。

> 联系第14讲实践 3：你实测过 group 越小量化误差越小。NF4 的 group=64 就是这个原理的应用。

## 三、Double Quantization：对 scale 再量化（省显存）

QLoRA 的另一创新。group_size=64 意味着每 64 个权重存一个 scale（fp32，4 字节）。对大模型，这些 scale 累计也占可观显存。

**Double quantization**：把 scale 本身也量化（8bit），省 4×→1.5× 显存。

```
原始：64 个权重 → 1 个 scale (fp32, 4B) + 64 个 4bit 值 (32B) = 36B
double quant：1 个 scale → 量化成 8bit (1B) + 次级 scale → 32B + 1B + 0.5B ≈ 33.5B
```

每 64 个权重省 ~2.5B，对 70B 模型累计省几个 GB。代价是引入二阶量化误差（scale 的量化误差），但实测影响很小。

vLLM 加载时 `bnb_4bit_use_double_quant=True`（`bitsandbytes_loader.py` 传给 bitsandbytes）开启。

## 四、NF4 vs 其他 4bit 格式对比

| 格式 | 量化点分布 | 设计依据 | 精度 | 适用 |
|------|-----------|---------|------|------|
| **INT4（均匀）** | 均匀等间距 | 简单 | 一般 | 通用 |
| **NF4** | 正态分位数 | LLM 权重正态分布 | 高 | QLoRA 训练 |
| **FP4（浮点）** | 浮点格式（指数+尾数）| 动态范围大 | 中 | 推理 |
| **AWQ/GPTQ（INT4 + 校准）** | 均匀 + 激活感知校准 | 保护重要权重 | 高 | 推理 |

**关键区别**：NF4 是"数据感知"（知道权重分布）但**无校准**（不用激活数据）；AWQ/GPTQ 是"校准感知"（用激活数据找重要权重）。两种思路正交。这也是第14讲说"训练用 NF4、推理可用 AWQ"的深层原因——它们优化目标不同。

## 五、NF4 的代价与边界

1. **依赖权重正态分布假设**：NF4 的量化点是按正态分布摆的。如果某层权重严重偏离正态（如某些激活、Embedding 层），NF4 精度优势消失甚至变差。QLoRA 只对 FFN/Attention 的线性层用 NF4（这些层权重确实近正态）。
2. **无校准**：NF4 不看激活，不知道哪些权重重要。相比 AWQ（保护高激活通道），NF4 可能损失关键权重精度。这也是推理时 AWQ 常优于直接用 NF4 的原因。
3. **bitsandbytes 依赖**：NF4 实现在 bitsandbytes 库，vLLM 只是调用方。bitsandbytes 的 kernel 性能不如 vLLM 原生量化 kernel（AWQ/GPTQ/FP8 的 Marlin/Machete）。所以 NF4 推理速度通常不如 AWQ。

> **工程结论**：NF4 是**训练**（QLoRA）的最优 4bit 格式（精度高、省显存、可微）。**推理**时，如果追求速度，往往把训练完的模型转成 AWQ/GPTQ（用更快的 kernel）。这正是第14讲 Q&A 讨论的"训练-推理量化不一致"的根源。

## 八、干中学实践任务（核心！）

> 在 `practice_nf4.py` 里实现 NF4 量化并对比均匀 INT4。
> 依赖：仅标准库（`math`, `random`, `statistics`）。不需要装 vllm/bitsandbytes。
> 设计哲学：你不只实现 NF4，还要**实测它为什么比均匀精度高**——生成正态分布权重，对比两种量化方法的误差。

### 实践 1：实现均匀 INT4 量化（热身）
实现 `uniform_quantize(values, bits=4)`：
- 找 max_abs，归一化到 [-1,1]
- 均匀摆 16 个点（等间距），每个值找最近点，存编号
- `uniform_dequantize(codes, scale)`：反量化

验证：round-trip 误差存在；量化点确实是等间距。

### 实践 2：实现 NF4 量化 + 对比（核心）
实现 `nf4_quantize(values)`：
- 用本讲第二节的 16 个 NF4 量化点（硬编码常量）
- 归一化 + 找最近 NF4 点 + 存编号
- `nf4_dequantize(codes, scale)`：反量化

**对比实验**（灵魂）：
1. 生成 10000 个**正态分布**权重（`random.gauss(0, 0.3)`）
2. 分别用均匀 INT4 和 NF4 量化→反量化
3. 测量平均量化误差（MSE）
4. **验证 NF4 误差 < 均匀误差**（因为权重正态，NF4 量化点匹配分布）

### 实践 3：分布不匹配的失败案例（进阶）
生成**均匀分布**权重（不是正态），重复实践2的对比。
- **验证此时 NF4 优势消失甚至变差**（因为 NF4 假设正态，权重均匀时它的"密集区"反而用不上）
- 这解释了为什么 NF4 只对正态分布的 LLM 权重有效

> 💡 实践 2 是灵魂。要点：① NF4 的 16 个点是固定常量（第二节）② 归一化让权重适配 [-1,1] ③ 找最近点 ④ 正态分布权重下 NF4 必然比均匀准。实践 3 让你亲手看到"数据感知量化的前提是数据确实匹配假设"——这是量化研究的核心洞察。
